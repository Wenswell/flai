# Brainstorm Phase

- 澄清目标、约束和成功标准。
- 信息不足时先提问，再给方案。
- 多个问题互不依赖时，一次性提出。
- 复杂问题先按该领域最强实践推理，再选择方向。
- 存在明显更简单的合理方向时，优先选择。
- 在 `.flai/conversation.md` 中保持 goal、conclusions、open questions 和 next step。
- 这些字段变化时，本轮结束前更新 `.flai/conversation.md`。
- 不要把原始聊天记录复制进 `.flai/conversation.md`。
- normal/deep 工作在实现前创建或更新任务 `plan.md`。
- 除非用户明确要求跳过规划，否则不要写生产代码。
