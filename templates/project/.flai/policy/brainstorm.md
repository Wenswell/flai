# Brainstorm Phase

- Clarify goal, constraints, and success criteria.
- Ask for missing information before proposing a plan.
- When questions are independent, ask them together.
- For complex questions, reason from strong domain practice before choosing a direction.
- Prefer the simplest reasonable direction when one exists.
- Keep goal, conclusions, open questions, and next step in `.flai/conversation.md`.
- Update `.flai/conversation.md` before ending when those fields changed.
- Do not copy chat transcripts into `.flai/conversation.md`.
- For normal/deep work, create or update task `plan.md` before implementation.
- Do not write production code unless the user explicitly asks to skip planning.
