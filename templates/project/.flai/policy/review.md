# Review Phase

- Review for bugs, regressions, missing verification, and skipped requirements.
- Use task `review.md` when present.
- Before finishing, check whether touched files should be simplified, merged, split, or removed.
- Remove clearly unnecessary files.
- Record unresolved risks in `status.md` or `.flai/conversation.md`.
- Do not finish normal/deep work while review issues remain.
- After code/file changes, state the commit message.
- If the current directory is a git repo, commit only the files changed for this task.
